# GUARD Act Would Mandate ID Checks for Basic Internet Use, EFF Warns

**TL;DR:** The GUARD Act, moving quickly through Congress, would require age verification for AI-powered tools, effectively mandating ID checks for search engines, homework help, and customer service—eliminating anonymous internet use.

## Key Points

- GUARD Act frames itself as targeting dangerous AI companions but broadly defines 'AI chatbot' to include most modern online tools
- Bill would require 'reasonable age verification' beyond simple checkboxes—likely government ID or biometric data for all users
- Minors would be blocked from AI-powered homework help, customer service chats, and search engines using AI
- Companies face up to $100,000 per violation with vague definitions like 'emotional interaction' creating overblocking incentives
- Age verification requirements would eliminate anonymous/pseudonymous internet use for basic tools
- EFF warns this creates surveillance infrastructure affecting everyone, not just minors
- Compliance costs would favor large companies over smaller developers, increasing tech consolidation

## Deep Dive

## Context and Scope Creep

The GUARD Act represents a classic pattern in internet regulation: legislation marketed as addressing specific harms (dangerous AI companions) but written so broadly it captures everyday tools. The bill's definition of 'AI chatbot' covers any system generating non-pre-written responses, which includes virtually all modern AI-powered services. The 'AI companion' category, from which minors are banned entirely, includes any chatbot designed to 'encourage or facilitate interpersonal or emotional interaction'—a description that could apply to homework helpers saying 'good question' or customer service bots expressing empathy.

## Technical and Privacy Implications

The 'reasonable age verification' requirement would fundamentally alter internet architecture by mandating identity verification for basic services. This creates several technical and privacy challenges: companies must implement ID verification systems tied to individuals, maintain ongoing verification (not one-time checks), and either build costly compliance infrastructure or outsource to third-party verification services while remaining legally liable. The requirement effectively eliminates anonymous internet use for AI-powered tools, which increasingly includes search engines and basic web services.

## Threat Model Analysis

From a freedom tech perspective, this represents infrastructure for comprehensive internet surveillance disguised as child protection. The law would normalize ID requirements for basic internet use, create databases of identity information tied to online activity, and establish precedent for age-gating based on vague content categories. The enforcement mechanism—$100,000 fines with unclear definitions—incentivizes overblocking and pushes companies toward restrictive compliance rather than nuanced implementation.

## Strategic Implications for Builders

This legislation, if passed, would create significant barriers for privacy-preserving services and anonymous access tools. Builders should consider how age verification mandates could be extended to other categories of online services, the compliance burden on smaller platforms versus large tech companies, and the precedent this sets for identity verification requirements in other contexts. The vague definitions and steep penalties suggest companies will choose broad blocking over narrow compliance, potentially affecting tool availability even for adults.
