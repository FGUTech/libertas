# Kimwolf Botnet Sybil Attack Disrupts I2P Anonymity Network

**TL;DR:** The massive Kimwolf IoT botnet accidentally crippled I2P's anonymous communications network by flooding it with 700k infected devices attempting to use it as fallback C2 infrastructure.

## Key Points

- Kimwolf botnet attempted to join 700,000 infected IoT devices to I2P network as backup command and control infrastructure
- Mass influx overwhelmed I2P's ~15,000-20,000 daily users, causing widespread service disruptions since Feb 3
- Attack constitutes a Sybil attack - single entity disrupting peer-to-peer network with massive fake identities
- I2P network operating at half capacity, with new stability improvements rolling out
- Botmasters acknowledged accident in Discord channel, seeking takedown-resistant C2 alternatives
- Similar experiments reportedly conducted with Tor network without major disruptions
- Incident highlights vulnerability of decentralized networks to resource exhaustion attacks
- Botnet recently lost competent operators, leading to rookie mistakes and 600k+ device losses
